# 基于Trie树与线段树的智能校园活动资源调度系统

## 项目概述

本项目旨在设计和实现一个智能校园活动资源调度系统，利用数据结构中的**Trie树**和**线段树**来优化活动参与者信息检索和资源冲突检测。该系统能够快速查询活动参与者、实时检测时间和资源冲突，提高校园活动的管理效率和资源利用率。

## 项目背景

随着大学校园活动的日益丰富，传统活动管理系统面临两大核心挑战：
1. 参与者检索效率低：现有系统查找参与者需遍历所有记录，无法快速定位。
2. 资源调度冲突频繁：活动安排依赖人工检查，难以避免时间、场地等资源冲突。

因此，设计一个支持快速检索和智能冲突检测的资源调度系统具有重要意义。

## 项目目标

1. **基于Trie树的前缀检索系统**：支持姓名/学号快速查询。
2. **基于线段树的资源冲突检测系统**：支持实时活动冲突检查。
3. **原子性文件持久化操作**：保证数据一致性，避免系统异常时的数据损坏。
4. **完整的校园活动管理前端界面**：提供数据可视化和用户交互功能。

## 技术方案

### 核心数据结构

- **Trie树**：用于联系人前缀检索，支持O(k)查询复杂度，其中k为查询前缀的长度。
- **双向链表**：用于联系人管理，支持O(1)的插入和删除操作。
- **线段树**：用于检测时间和资源的冲突，支持O(log n)的查询和更新操作。
- **哈希表**：用于快速检查联系人信息的去重，支持O(1)操作。

### 主要算法

1. **Trie树前缀匹配算法**：姓名/学号快速检索，查询复杂度O(k)。
2. **线段树区间查询算法**：用于时间冲突检测，查询复杂度O(log n)。
3. **文件持久化原子操作**：通过写临时文件、校验和重命名，保证数据一致性。
4. **双向链表快速插入删除**：支持高效的联系人管理。

## 系统要求

1. **功能要求**：
   - 支持联系人前缀检索（姓名/学号），响应时间<100ms。
   - 支持活动时间、场地冲突实时检测。
   - 文件操作具备原子性，异常重启后数据可恢复。
   - 支持操作撤销/重做功能。

2. **性能要求**：
   - Trie树检索性能比线性搜索提升10倍以上。
   - 线段树冲突检测性能比暴力遍历提升10倍以上。
   - 系统支持至少5000个联系人和1000个活动的高效管理。

3. **测试要求**：
   - 单元测试覆盖率≥60%。
   - 完成5组以上检索性能对比实验。
   - 完成冲突检测性能对比实验，完成系统集成测试。
